# Release Notes — 2026-02-24

## OpenAI OAuth Subscription Sign-In

Users with a ChatGPT Plus/Pro subscription can now sign in with their OpenAI account instead of pasting an API key.

- **PKCE OAuth 2.0 flow** via `ASWebAuthenticationSession` against `auth.openai.com`
- **Token refresh** — access tokens are auto-refreshed before expiry using the stored refresh token
- **Model catalog** — after sign-in, available models are fetched and a preferred model is auto-selected
- **JWT account ID extraction** — Codex-style requests inject the ChatGPT account ID header
- **Auth mode picker** — provider editor shows API Key / OpenAI Sign-In toggle for OpenAI-compatible providers

## OpenAI Responses API via WebSocket

OpenAI-compatible providers now support the Responses API over WebSocket for lower-latency streaming.

- **WebSocket transport option** in provider editor (HTTP or WebSocket)
- **`responses_websockets=2026-02-06` beta** header for OpenAI Responses WebSocket endpoint
- **Codex Responses HTTP fallback** with `responses=experimental` beta header
- **Tool-calling requests** still use HTTP chat/completions (WebSocket tool support pending)
- **iOS standalone transport** — `OpenAIWebSocketChatTransport` for direct WS chat on iOS

## Dream Mode — End-to-End Improvements

### Dedicated Dream Journal Session

Dream chat now runs in a separate `dream-journal` session instead of polluting the main chat.

### Configurable Dream Settings

Three new pickers in Settings > Device > Dream Mode:

| Setting | Options | Default |
|---------|---------|---------|
| **Dream Iterations** | 6, 8, 10, 12, 16, 20 | 12 |
| **Reasoning Level** | Off, Low, Medium, High | Medium |
| **Dream Model** | Default (current) + all configured providers | Default |

### Dream Provider Switching

When a dedicated dream model is selected, the runtime automatically switches to that provider before the dream cycle and restores the previous provider after it exits.

### Digest Action Triage

Dream digest now classifies verify-next items by risk:

- **`[LOW-RISK]`** — read-only actions (fetch URLs, search, read files) are auto-executed on wake
- **`[HIGH-RISK]`** — write actions (modify files, API calls with side effects) are listed for user approval

### Clean Dream Journal Output

After each dream cycle, a readable summary with timestamp and proposed actions is posted to the dream-journal session (with tools disabled to avoid markup noise).

### Heartbeat Noise Removed

- Dream prompt no longer references HEARTBEAT.md — LLM reads DREAM.md directly
- Removed `get_idle_time()` call from dream cycle (saves a tool round)
- Removed `HEARTBEAT_OK` response instruction
- HEARTBEAT.md dream section updated to match

### Cooldown Removed

- 4-hour cooldown has been removed entirely — dream triggers whenever idle exceeds threshold
- Interaction epoch tracking still prevents duplicate dreams for the same idle period
- **Enter Dream Now** button in Settings for manual triggering

### Repeating Dreams During Continuous Idle

- Dream now re-triggers every `idleThresholdSeconds` during continuous idle (no user interaction required)
- Old epoch-based blocking replaced with time-since-last-dream check using `lastRunAt`
- If the last dream started less than the threshold ago, auto-trigger is skipped; otherwise dream fires again

## What to Test

1. **OpenAI OAuth** — Add a new OpenAI-compatible provider, switch auth to "OpenAI Sign-In", tap "Sign in with OpenAI (Web)". Verify the browser opens, you can sign in, and models are loaded after redirect.
2. **Token refresh** — After signing in, wait for the access token to near expiry and verify it auto-refreshes without re-prompting.
3. **WebSocket transport** — Configure an OpenAI provider with WebSocket transport. Send a message and verify streaming response arrives.
4. **Dream journal isolation** — Trigger a dream. Verify output appears in `dream-journal` session, not main chat.
5. **Dream settings** — Change iterations, reasoning level, and model in Settings. Trigger a dream and verify the LLM uses the configured values.
6. **Digest triage** — After a dream exits, verify the digest in main chat shows LOW-RISK/HIGH-RISK tags and the LLM auto-executes low-risk items.
7. **Enter Dream Now** — Tap the button in Settings. Verify dream starts immediately.
8. **Clean journal output** — Open dream-journal session after a dream. Verify readable summary with timestamp, not raw tool-call markup.
