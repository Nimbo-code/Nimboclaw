# Release Notes — 2026-02-25

## Microphone Dictation in Chat Composer (iOS)

A new microphone button in the chat toolbar enables speech-to-text dictation directly into the message input field.

- **On-device speech recognition** via `SFSpeechRecognizer` — works offline when supported
- **Mic button** in the toolbar row (between the thinking-level picker and the refresh button)
- **Live transcription** — partial results stream into the input field as you speak
- **Tap to toggle** — tap once to start dictation, tap again to stop; the transcribed text remains editable before sending
- **Auto-scroll** — long dictated text automatically scrolls to keep the latest words visible
- **Send clears dictation** — tapping send stops any active dictation and clears the input field
- **Voice-wake coordination** — dictation suspends the always-on voice-wake listener while recording and resumes it when done
- **Protocol-based injection** — `ChatDictationProvider` protocol in the shared `OpenClawChatUI` package keeps Speech/AVFoundation dependencies out of the UI layer; concrete `ComposerDictationManager` lives in the app target
- **Background-thread audio pipeline** — audio engine setup and `SFSpeechRecognizer.recognitionTask()` run on a background queue to avoid `_dispatch_assert_queue_fail` in the Speech framework's internal queues

### UI

- Idle state: `mic` icon with bordered button style (matches existing toolbar buttons)
- Recording state: `mic.fill` icon with red tint and subtle pulse animation
- Hidden on macOS and tvOS (`#if os(iOS)`)

## OpenAI Responses API — SSE Streaming

- Switched OpenAI Responses API transport from non-streaming to **Server-Sent Events (SSE) streaming**
- Updated request headers and payload for streaming mode
- Improved error handling for streamed response chunks
- Updated response parsing to handle incremental SSE data

## Device Tool Permissions (iOS)

- Enabling a device tool (Calendar, Contacts, Reminders, etc.) now **requests the corresponding iOS permission** immediately on toggle-on, instead of deferring to first use
- Added missing `NSRemindersFullAccessUsageDescription` and `NSCalendarsFullAccessUsageDescription` privacy keys to Info.plist

## OpenAI Responses API — Tool Support for Codex/ChatGPT Backend

Previously, when using the ChatGPT (Codex) backend (`chatgpt.com/backend-api/codex/responses`), the `completeWithTools()` path stripped all tool definitions and sent a plain text completion — the LLM never received any tools.

- **Full Responses API tool calling** — tool definitions are now properly formatted and included in Codex backend requests (`type: "function"`, `name`, `description`, `parameters`)
- **Tool call extraction** — `extractOpenAIResponsesToolCalls()` parses `function_call` items from the Responses API output array
- **Multi-turn tool messages** — `mapOpenAIMessagesToResponsesInput()` now maps `tool` role messages to `function_call_output` items and `assistant` tool calls to `function_call` items
- **Tool name normalization** — dots converted to underscores for OpenAI compliance (e.g., `camera.snap` → `camera_snap`) with reverse mapping for local dispatch

## Image Vision in Tool Results

When `camera.snap` or `photos.latest` return base64 image data, the image is now sent to the LLM as a proper multimodal content block instead of raw base64 text (which was being truncated by the 8000-char clamp).

- **Image extraction** — `extractImageFromPayload()` strips base64 from camera/photo tool results and replaces with `[image attached]` placeholder
- **Server-side resize** — `resizeBase64JPEG()` downscales images to 512px max width using CoreGraphics/ImageIO (no UIKit dependency), re-encodes at 0.7 JPEG quality
- **Multimodal content blocks** — images sent as `image_url` data URIs for Chat/Completions API and `input_image` blocks for Responses API
- Supports both `camera.snap` (top-level `base64` key) and `photos.latest` (nested `photos[0].base64`)

## Camera Snap → Composer Attachment

When the LLM calls `camera.snap`, the captured photo now appears as an attachment in the chat composer UI.

- **Observable publish** — `TVOSLocalGatewayRuntime.lastCameraCapture` property updated via `DeviceToolBridgeImpl.onCameraCapture` callback
- **SwiftUI observer** — `ChatSheet` watches `lastCameraCapture` via `.onChange` and calls `viewModel.addImageAttachment()`
- Attachment appears with thumbnail, can be removed, and is included in the next sent message

## Diagnostic Logging

Added trace logging throughout the LLM tool pipeline for debugging tool delivery and image handling.

- `[GatewayLocalLLM]` logs in `completeWithTools()` — tool names, message count, image presence
- `[GatewayLocalLLM]` logs in `completeViaOpenAIResponsesHTTP()` — request body size, tool count, result details
- `[GatewayLocalMethodRouter]` logs in image extraction — payload keys, base64 length, resize results
- `[GatewayLocalMethodRouter]` logs in `mapOpenAIMessagesToResponsesInput()` — `input_image` block emission

## Bug Fixes

- **Fixed contacts search crash** — `CNPropertyNotFetchedException` when using `CNContactFormatter.string(from:style:.fullName)` without the required key descriptor; added `CNContactFormatter.descriptorForRequiredKeys(for: .fullName)` to `payloadKeys`
- **Fixed keyboard/mic conflict** — tapping the mic now dismisses the on-screen keyboard first; if the keyboard appears while dictating, dictation stops automatically
- **Fixed mic button layout oscillation** — replaced `scaleEffect` + `repeatForever` animation with `symbolEffect(.pulse)` to prevent the button from shifting left/right on state toggle
- Fixed thinking-level picker (Low/Medium/High) getting truncated when the mic button is present — picker now uses fixed intrinsic size
- Fixed dream auto-trigger using epoch blocking — replaced with time-based re-trigger via `lastRunAt`
- Fixed build error: use `dreamManagerRef` (not `dreamManager`) in heartbeat skip check

## What to Test

1. **Dictation** — Tap the mic button in the chat composer toolbar. Grant microphone and speech recognition permissions. Speak and verify text appears in the input field in real time.
2. **Long dictation scroll** — Dictate a long message. Verify the input field scrolls to show the latest text.
3. **Send after dictation** — Dictate a message, then tap send. Verify the message is sent and the input field clears completely.
4. **Stop dictation** — Start dictation, then tap the mic button again. Verify dictation stops and the partial text remains editable.
5. **Voice-wake resume** — If voice wake is enabled, start dictation then stop. Verify voice wake resumes listening.
6. **SSE streaming** — Send a message via an OpenAI-compatible provider. Verify the response streams in with SSE.
7. **Device tool permissions** — Go to Settings > Tools, enable Calendar or Contacts. Verify the iOS permission dialog appears immediately.
8. **Thinking picker layout** — Verify the thinking-level picker (Low/Medium/High/X-High) displays its label on a single line alongside the mic button.
9. **Contacts search** — Use the contacts tool to search for a contact. Verify no crash and results return correctly.
10. **Keyboard/mic mutual exclusion** — Start dictation, then tap the text input to bring up the keyboard. Verify dictation stops. Conversely, with the keyboard showing, tap the mic. Verify the keyboard dismisses.
11. **Codex/ChatGPT tool calling** — Connect via a ChatGPT 5.2 backend. Ask the agent to use a device tool (e.g., "take a photo"). Verify the tool is called and the result is returned.
12. **Camera snap image vision** — Ask the agent to take a photo and describe it. Verify the agent receives the image and can describe its contents (not just metadata).
13. **Camera snap composer attachment** — After the agent calls `camera.snap`, verify the captured image appears as an attachment thumbnail in the composer strip.
14. **Photos tool image vision** — Ask the agent to look at your latest photo. Verify the image is sent as a multimodal block and the agent can describe it.
