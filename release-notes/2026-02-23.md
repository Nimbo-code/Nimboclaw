# Release Notes — 2026-02-23

## Dream Mode — Artifact Pipeline & Heartbeat Integration

Dream Mode now has a full artifact pipeline that connects idle-time dreaming to real outputs the user can review on return.

### Dream Artifacts Directory

A new `dream/` directory under the workspace root provides a deterministic, prunable location for all dream outputs — separate from `memory/` to avoid polluting grounded recall with speculative content.

```
dream/
├── state.json          — run bookkeeping (managed by native code)
├── digest.md           — short wake summary (overwritten each run)
├── journal/
│   └── YYYY-MM-DD.md   — verbose audit trail (14-day retention)
├── patches/
│   └── *.patch          — proposed diffs, never auto-applied (7-day retention)
└── archive/             — optional compressed old journals
```

### DreamStateStore

A new `DreamStateStore` persists `dream/state.json` atomically with `NSLock`-based thread safety. It tracks:

- **Run bookkeeping** — `lastRunId`, `lastRunAt`, `pendingDigestPath`
- **Cooldown** — `cooldownUntil` prevents re-entering dream mode within 4 hours of the last run
- **Interaction epoch** — `lastDreamForInteraction` prevents duplicate dreams for the same idle period
- **Digest delivery** — `deliveredForInteraction` ensures each digest is delivered exactly once

### Heartbeat Integration

The heartbeat/cron system now drives dream mode end-to-end:

- **Auto-trigger with cooldown** — When the user is idle past the threshold, the native auto-trigger enters dream mode only if the cooldown has expired and the interaction epoch is new.
- **Digest delivery on return** — When the user comes back (idle < 5 minutes), `evaluateDigestDelivery` flags the pending digest for the next heartbeat turn, which reads `dream/digest.md` and delivers it as a chat message.
- **HEARTBEAT.md template** updated with two concrete behaviors: (A) start dream when idle, (B) deliver digest when user returns.
- **DREAM.md template** updated with artifact directory layout, write rules (`write` tool, not `memory.append`), digest format spec, and retention policy.

### Extended Tool Responses

- **`dream_mode`** — Response now includes `runId` (current dream run UUID), `outputRoot` ("dream"), and `writeMode` ("patches"). On exit, triggers automatic retention cleanup.
- **`get_idle_time`** — Response now includes `last_interaction_at` (ISO 8601), `pending_digest_path`, and `cooldown_until`, giving the heartbeat LLM all information needed to decide whether to start or deliver dreams.
- **Tool definitions** updated with optional `outputRoot` and `writeMode` parameters.

### Retention & Cleanup

- **Age-based cleanup** runs on dream exit and app launch:
  - Journal entries (`dream/journal/*.md`) older than 14 days are deleted
  - Patches (`dream/patches/*.patch`) older than 7 days are deleted
- **Memory isolation** — `dream/` is intentionally excluded from memory search and `memory.append` path validation.

### Chat UI — Tool Display

- `dream_mode` tool calls now show `runId` and `outputRoot` in the tool call detail
- `get_idle_time` tool calls now show `pending_digest_path` alongside existing fields

## Dream Mode — Animations & Visual Overlay

Six screensaver-style animations are available in **Settings > Device > Dream Mode**:

| Animation | Description |
|-----------|-------------|
| **Flame Pulse** | Warm radial gradient that breathes in and out (default) |
| **Aurora** | Slow-moving northern-lights colour bands |
| **Starfield** | Classic forward-flying star tunnel |
| **Breathing Orb** | Soft pulsing circle with glow |
| **Flurry** | Particles drifting through a gradient field |
| **Flurry Classic** | macOS-style Flurry ribbons with Classic Mac OS colour palette |

### Settings UI

- **Live thumbnail grid** — Each animation tile shows a real-time miniature preview
- **Large preview** — Tapping a tile selects it and updates a larger preview above the grid
- **Idle threshold** — Configurable in seconds (default: 600s / 10 minutes)
- **Enable toggle** — Master switch for automatic dream activation

### Idle Tracking

- `UserIdleTracker` monitors touch events via `IdleTouchPassthroughView`
- Dream overlay appears automatically after the configured idle threshold
- Any touch interaction dismisses the overlay and resets the idle timer

### LLM Tool Access

- **`dream_mode`** tool — enter/exit/status actions for programmatic control
- **`get_idle_time`** tool — returns current idle seconds, dream state, and dream-enabled flag

## Bootstrap Budget — Redundant Clamping Cleanup

Removed redundant `max(1000, …)` / `max(4000, …)` clamping from three layers (UI onChange, admin API parser, UserDefaults loader). Clamping now happens at a single point in `normalizedSettings()`, which all code paths already flow through.

## What to Test

1. **Dream artifact pipeline** — Enable Dream Mode, wait for idle trigger (or use a short threshold). Verify `dream/state.json` is created in the workspace with correct fields (`lastRunId`, `lastRunAt`, `cooldownUntil`).
2. **Cooldown** — After a dream exits, verify it does not re-trigger within 4 hours even if idle threshold is met again.
3. **Tool responses** — In chat, ask the LLM to call `dream_mode(enter)` and verify the response includes `runId` and `outputRoot`. Call `get_idle_time` and verify it includes `pending_digest_path` and `cooldown_until`.
4. **Digest delivery** — After a dream run completes, return to the app (interact) and verify the heartbeat delivers the digest as a chat message on the next turn.
5. **Retention cleanup** — Create test files in `dream/journal/` with old modification dates; trigger a dream exit and verify old files are cleaned up.
6. **Memory isolation** — Verify `memory.search` does NOT return results from the `dream/` directory.
7. **Animations** — Go to Settings > Device > Dream Mode. Scroll through all 6 animation tiles — each should show a live thumbnail preview. Tap each to select, verify the larger preview updates.
8. **Idle overlay** — Set a short threshold (e.g., 30s), wait, and verify the dream overlay appears. Touch to dismiss.
