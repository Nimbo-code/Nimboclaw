# Release Notes — 2026-02-22

## Skill Credentials System

Skills that require API keys (Notion, Trello, etc.) can now store and retrieve them securely through the iOS Keychain. Three new device tools handle the lifecycle:

- **`credentials.get`** — check whether a key exists and retrieve it
- **`credentials.set`** — store a key in the Keychain
- **`credentials.delete`** — remove a stored key

When `credentials.get` returns `hasKey: false`, a blue **"Set up API key"** button appears automatically in the chat UI. Tapping it opens a secure entry sheet — the user never needs to paste keys into the conversation.

The bootstrap prompt (AGENTS.md) now includes a full credential flow guide so the LLM follows the button-based onboarding automatically.

## Tools Settings

A new **Settings > Tools** section lists every registered tool with its category and description. Individual tools can be enabled or disabled at runtime, letting users reduce noise for workflows that don't need certain capabilities.

## Skills Management Improvements

- **Delete skills** — Skills can now be removed from Settings > Skills (disable first, then delete).
- **Copy skill content** — A copy button in the skill inspector copies the full SKILL.md to the clipboard.
- **Credential status in skill inspector** — The skill detail sheet shows which credentials the skill needs and whether they are configured.
- **New built-in skills** — `x-twitter-api-search` and `github` skill templates are now included in the bootstrap template store.
- **Skill creation guidance** — The bootstrap prompt now instructs the LLM to always write new skills to `skills/<name>/SKILL.md` and verify placement with `ls`.

## Workspace `ls` Tool

A new `ls` tool lets the LLM list files and directories inside the workspace, with optional recursive mode. Paths are sandboxed to the workspace root. Capped at 500 entries to prevent oversized responses.

## Bootstrap Prompt Budget Controls

- **Configurable limits** — `perFileMaxChars` and `totalMaxChars` for bootstrap-injected files are now exposed in Settings and can be tuned per-device.
- **Increased defaults** — Per-file limit raised from 5,000 to 8,000 chars; total budget raised from 16,000 to 48,000 chars.
- **Dropped-file tracking** — Files that exceed the budget are recorded and accessible via `lastBootstrapDroppedFiles` for diagnostics.

## Chat UI Enhancements

- **Copy message button** — Every chat bubble now shows a subtle copy icon on hover/tap that copies the message text to the clipboard.
- **Clear conversation** — A "Clear Conversation" option in the composer menu permanently deletes all messages in the current session.
- **Continue button** — The refresh button in the composer is now a "Continue" action that sends "Continue" to the LLM, useful for resuming interrupted tool-calling loops.
- **Credential prompt cards** — `credentials.get` results with `hasKey: false` render as visible prompt cards even when "Show Tool Calls" is turned off.

## WebSocket Server Resilience

- **LAN fallback** — If binding to a LAN interface fails, the server now automatically falls back to localhost so on-device chat keeps working. A `isFallbackLoopback` flag exposes the state.
- **Listener state recovery** — A `onListenerStateChange` callback lets the runtime detect and restart the listener after network interface changes.

## LLM Request Telemetry

- **Request body size tracking** — `requestBodyBytes` is now captured on every LLM API call (OpenAI-compatible and Anthropic) and accumulated across tool-calling loops.
- **Per-chat token and round-trip stats** — `ChatExecutionResult` now reports `totalInputTokens`, `totalOutputTokens`, `totalRequestBodyBytes`, and `llmRoundTrips`.

## Documentation

- New [credentials.md](../doc/credentials.md) developer doc covering architecture, Keychain naming, UI integration, and security considerations.
- Root README updated to clarify that Nimboclaw runs entirely inside the app sandbox with no external server.
